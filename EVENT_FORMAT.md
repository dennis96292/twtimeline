## 格式

輸出必須是一個 **JSON 陣列**，每個元素代表一筆歷史事件。

```json
[
  {
    "date": "YYYY-MM-DD",
    "title": "事件標題",
    "desc": "事件描述",
    "tags": ["標籤一", "標籤二"],
    "source_url": "https://..."
  }
]
```

---

## 欄位說明

| 欄位 | 型別 | 必填 | 說明 |
|------|------|------|------|
| `date` | string | 是 | 事件發生日期，格式 `YYYY-MM-DD`。若僅知年份填 `YYYY-01-01`，僅知年月填 `YYYY-MM-01` |
| `title` | string | 是 | 簡短事件名稱，建議 20 字以內 |
| `desc` | string | 否 | 事件描述，建議 50～150 字，說明事件背景、經過與影響 |
| `tags` | array | 否 | 分類標籤陣列，從下方標準標籤中選取，可複選 |
| `source_url` | string | 否 | 新聞或資料來源網址，無則填空字串 `""` |

---

## 標準標籤

請從以下標籤中選取，**不要自創新標籤**：

- `政治`
- `選舉`
- `兩岸`
- `外交`
- `經濟`
- `社會`
- `人權`
- `災害`
- `疫情`
- `環境`
- `歷史`

---

## 規則

1. 整體輸出只能是 JSON 陣列，**不附加任何說明文字、markdown 區塊、程式碼區塊標記**
2. 所有字串值使用**繁體中文**
3. `date` 必須是合法日期，年份範圍 `1895-01-01` ～ 今日
4. `title` 不可與同一批資料中其他事件重複
5. `tags` 陣列不可為 `null`，無標籤時填 `[]`
6. `source_url` 不可為 `null`，無來源時填 `""`
7. 若事件橫跨多日（如颱風、疫情），填**事件開始日**或**最具代表性的單一日期**

```json

    "date": "2024-01-13",
    "title": "2024總統大選",

    "tags": ["政治", "選舉"],
    "source_url": ""
  },
  {
    "date": "1999-09-21",
    "title": "921大地震",
    "desc": "凌晨1時47分，芮氏規模7.3強震襲台，震央位於南投縣集集鎮，造成2,415人罹難、逾10萬棟建物受損，為戰後台灣傷亡最慘重的天然災害。",

    "tags": ["歷史", "人權", "政治"],
    "source_url": ""
  }

```

## 常見錯誤（請避免）

| 錯誤 | 正確 |
|------|------|
| `"date": "1999/09/21"` | `"date": "1999-09-21"` |
| `"date": "民國88年9月21日"` | `"date": "1999-09-21"` |
| `"tags": null` | `"tags": []` |
| `"source_url": null` | `"source_url": ""` |
| 輸出包含 \`\`\`json 區塊標記 | 直接輸出純 JSON |
| `"title": "九二一大地震九二一大地震發生於1999年..."` | title 簡短，細節放 desc |
